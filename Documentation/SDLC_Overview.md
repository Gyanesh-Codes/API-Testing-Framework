# SDLC Overview

Software Development Life Cycle is the structured process used to plan, build, test, release, and maintain software. QA participates throughout the SDLC, not only at the end.

## Phases
1. Requirement Analysis: Understand functional and non-functional expectations.
2. Planning: Estimate scope, risks, tools, environments, and timelines.
3. Design: Review architecture, API contracts, data flow, and acceptance criteria.
4. Development: Developers implement features while QA prepares test scenarios and test data.
5. Testing: QA executes functional, API, smoke, regression, exploratory, and negative testing.
6. Deployment: Release is moved to staging or production after quality gates.
7. Maintenance: Defects, improvements, monitoring feedback, and regression risks are handled.

## Agile and Scrum
In Agile, work is delivered in short iterations called sprints. Scrum ceremonies include sprint planning, daily stand-up, backlog refinement, sprint review, and retrospective. QA contributes by clarifying acceptance criteria, designing tests early, automating repeatable checks, reporting defects, and supporting release decisions.
