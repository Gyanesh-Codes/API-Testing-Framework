# STLC Overview

Software Testing Life Cycle is the QA process followed to test software systematically.

## STLC Phases
1. Requirement Analysis: Identify testable requirements and risks.
2. Test Planning: Define scope, strategy, schedule, resources, tools, and entry/exit criteria.
3. Test Case Design: Create test cases, test scenarios, test data, and traceability.
4. Test Environment Setup: Prepare Postman environment variables, API keys, and test data.
5. Test Execution: Run tests, compare actual results with expected results, and record status.
6. Defect Reporting: Log defects with severity, priority, reproduction steps, and evidence.
7. Test Closure: Summarize execution, defects, risks, lessons learned, and sign-off status.

## Testing Types Used
- Functional Testing validates whether APIs meet expected behavior.
- Smoke Testing verifies critical APIs are stable enough for deeper testing.
- Regression Testing ensures existing workflows still work after changes.
- Exploratory Testing uses tester investigation to discover gaps not covered by scripted tests.
- Negative Testing validates error handling and input validation.
